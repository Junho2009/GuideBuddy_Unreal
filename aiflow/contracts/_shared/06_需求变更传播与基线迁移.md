# 需求变更传播与基线迁移

## 何时需要 Change Request

以下变化需要先写 Change Request：

- MVP 目标变化。
- 本阶段 in-scope / out-of-scope 变化。
- 验收口径变化。
- 输出文件结构变化。
- 自动验收能力变化。
- 当前 baseline 已 frozen 或 active 后又要调整。

## 迁移顺序

1. 写 Change Request。
2. 标记受影响合同为 `stale`。
3. 更新 specs 或说明 prose 不变。
4. 重编译 Brief / Task Pack / Manifest / Verifier。
5. 同步 issue 或 `CONTEXT.md` 的 baseline 指针。
6. 重新执行 verifier。
7. 写新的 run / evidence / ledger。

## 当前注意

当前 MVP01 仍是 `draft`，尚未 frozen。草案内部调整不需要 Change Request，但一旦切为 `frozen` 或 `active`，后续变更必须走 CR。
