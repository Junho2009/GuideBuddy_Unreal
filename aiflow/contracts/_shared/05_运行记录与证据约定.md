# 运行记录与证据约定

## Run

Run 记录 Verifier 实际执行过程，放在：

```text
aiflow/contracts/runs/
```

Run 不应只写“通过”，而应记录：

- 使用的 baseline。
- 执行时间。
- 执行环境。
- 每条检查的结果。
- 证据路径。
- 失败项与重跑项。

## Evidence

Evidence 分两层：

1. 原始证据：通常放在 `Saved/GuideBuddy/Telemetry/<run-id>/`。
2. 合同索引：放在 `aiflow/contracts/evidence/<baseline>/run-xxx/index.md`。

## Ledger

Ledger 只在验收通过并 accepted 后写入，放在：

```text
aiflow/contracts/ledgers/
```

没有 accepted run 时，不要提前写 result ledger。
