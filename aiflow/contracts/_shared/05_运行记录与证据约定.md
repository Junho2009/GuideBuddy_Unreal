# 运行记录与证据约定

## Run

Run 记录 Verifier 实际执行过程，放在：

```text
aiflow/contracts/runs/
```

Run 不应只写“通过”，而应记录：

- 使用的 baseline。
- 执行时间。
- 执行环境。
- 每条检查的结果。
- 证据路径。
- 失败项与重跑项。

## Evidence

Evidence 分两层：

1. 本地运行原始证据：通常先输出到 `Saved/GuideBuddy/Telemetry/<run-id>/`。
2. 版本化归档证据：通过 verifier 的关键 raw 文件复制到 `aiflow/contracts/evidence/<phase>/<baseline-version>/run-xxx/raw/`。
3. 合同索引：放在 `aiflow/contracts/evidence/<phase>/<baseline-version>/run-xxx/index.md`。

`Saved/` 目录不作为整体版本化对象；accepted 或 verifier-passed run 的关键 raw evidence 应随 evidence index 一起归档，避免索引指向已经消失的本地运行目录。

## Ledger

Ledger 只在验收通过并 accepted 后写入，放在：

```text
aiflow/contracts/ledgers/
```

没有 accepted run 时，不要提前写 result ledger。
