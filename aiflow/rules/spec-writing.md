# specs 写作规则

`aiflow/specs/` 是 GuideBuddy MVP 的 prose 需求层。

## 写什么

- 目标
- 范围
- 非目标
- 验收方式
- 依赖与衔接
- 阶段状态

## 不写什么

- 具体代码改法
- 编辑器点击步骤
- 临时排障过程
- Verifier run 结果
- 原始证据

## 推荐结构

```markdown
# MVPXX：名称

## 本阶段目标
## 本次一定要有
## 本次先不做
## 验收方式
## 依赖与衔接
## 当前状态
## 备注
```

## 更新时机

- 目标变化。
- 范围或非目标变化。
- 验收方式变化。
- 依赖关系变化。
- 阶段状态变化。

纯实现细节变化通常不改 specs。
